## Example Scenarios

### Scenario 1: Standard Admin Setup
```
Role: ADMIN
Modules: [BOOKING, CUSTOMER, MARKETING, FINANCE, TOOLS, USER_MANAGEMENT, REPORTS]
Custom: None
Result: Full admin access with default permissions
```

### Scenario 2: Limited Admin (Bookings Only)
```
Role: ADMIN
Modules: [BOOKING, CUSTOMER, TOOLS]
Custom: None
Result: Admin can manage bookings and customers, but no finance or marketing
```

### Scenario 3: Support with Temporary Finance Access
```
Role: SUPPORT
Modules: [BOOKING, CUSTOMER, TOOLS]
Custom: [
  {
    module: FINANCE,
    actions: [VIEW, EXPORT],
    expiresAt: "2025-12-22T10:00:00Z",
    reason: "Quarterly audit assistance"
  }
]
Result: Standard support + temporary finance view for 24 hours
```

### Scenario 4: Read-Only Analyst
```
Role: DEVELOPER (or custom role)
Modules: [BOOKING, CUSTOMER, FINANCE, REPORTS]
Custom: [
  {
    module: BOOKING,
    actions: [VIEW],
  },
  {
    module: REPORTS,
    actions: [VIEW, EXPORT],
  }
]
Result: Can view all modules, export reports only
```
